# WEB 프론트엔드 테스트 수행을 위한 기초 Vendor 프로젝트

별도의 `npm install` 없이 곧바로 사용할 수 있는 프로젝트입니다.

## 1. 목적

본 프로젝트는 배포 후 WEB 주요 기능의 회귀 검증을 자동화하기 위한 Playwright 기반 테스트 프로젝트입니다.

## 2. 사용 시점

- 보안 패치 배포 후
- 고도화 개발 반영 후
- 코드 품질 개선 후
- 운영 반영 전 사전 검증
- 야간 배포 후 기본 기능 확인

## 실행 환경

- **Node.js**: `24.15.0`
- **npm**: `11.12.1`

## 사용 방법

### 1. 테스트 녹화

아래 명령어로 테스트를 녹화할 수 있습니다.

```bash
npm run codegen -- -o tests/my_new_test.spec.js
```

- `-- -o ...` 옵션은 **테스트 파일을 자동 생성**하기 위한 옵션입니다.
- 자동 파일 생성을 원하지 않으면 해당 옵션은 제거해도 됩니다.

### 2. 테스트 실행

아래 명령어로 작성된 테스트를 실행할 수 있습니다.

```bash
npm run test
```

- 만들어 둔 `.js` 파일을 실행하여 테스트를 직접 수행합니다.
- 테스트 파일명은 반드시 아래 형식 중 하나여야 실행됩니다.
  - `*.test.js`
  - `*.spec.js`

### 3. 테스트 리포트 확인

아래 명령어로 테스트 결과 리포트를 브라우저에서 확인할 수 있습니다.

```bash
npm run report
```

- 테스트 결과를 브라우저에서 보기 좋게 정리된 형태로 확인할 수 있습니다.

## 요약

| 항목 | 명령어 |
|---|---|
| 테스트 녹화 | `npm run codegen -- -o tests/my_new_test.spec.js` |
| 테스트 실행 | `npm run test` |
| 리포트 확인 | `npm run report` |

## 참고

- 별도의 의존성 설치 없이 바로 사용 가능합니다.
- 테스트 파일명 규칙을 지켜야 정상적으로 실행됩니다.
